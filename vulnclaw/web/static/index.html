Fallback Web Shell

VulnClaw Web 已启动

后端 API 正在运行，但没有检测到已构建的 React 前端。此页面是轻量 ToC fallback，用于确认服务状态并提示如何进入完整图形界面。

需要构建完整前端

授权优先

输入目标，确认边界，再开始安全检查

完整 ToC Web 界面会提供首页检查向导、风险结果、报告中心、安全边界、历史和设置页。 如果你看到这个页面，说明后端回退到了内置静态页，而不是前端构建产物。

检查后端健康状态 查看公开配置
如何进入完整界面

构建 React 前端

在项目目录执行:

cd frontend

npm install

npm run build

然后重新打开 vulnclaw web 提供的地址。

ToC 主流程

用户路径

  1. 1输入明确授权的 URL、域名或 IP。
  2. 2选择快速摸底、标准检查或深度验证。
  3. 3确认端口、主机、路径和动作边界。
  4. 4查看风险结果，并导出 Markdown / HTML 报告。
安全边界

不要越过授权范围

VulnClaw 的完整 Web 界面会把 only_*、blocked_* 和动作约束展示成可读边界，并在发生越界尝试时进入安全边界页查看拦截原因。

建议普通自测至少指定主机、端口或路径之一。

如果你已经构建过前端但仍看到此页，请确认 frontend/dist/index.html 存在，并重启 Web 服务。
